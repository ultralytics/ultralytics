# SC-ELAN 突破点（创新点）合理性复核与下一步研究约束（基于 `sc-elan.md`）

日期：2026-02-26  
范围：仅基于仓库内 `sc-elan.md` 的设计描述、变体定义与 VisDrone2019-DET-test-dev（含 val4 批次）结果总结进行复核与约束化整理。

---

## 1. 结论摘要（可直接引用）

- **总体判断：创新点“方向合理、工程可落地”，但目前更像“强组合方案的增量式创新”，需要用更严格的消融与统计鲁棒性把“不可替代性”讲清楚。**
- **结构主线已基本收敛**到 `LSKA + TSCG`，并在 val4 中达到当前顶线 **mAP50-95 = 0.210**（`LSKA23-TSCG` 与 `DetectCAI-Soft` 并列）。
- **检测头 CAI 属于“收益依赖策略、存在回退风险”的训练策略族**：`Soft` 稳定、`Mom098` 回退，说明必须对其探索空间施加约束，否则容易出现“看似创新、实际不稳定”的不可发表风险。
- **下一步不应继续无约束叠模块**。必须以“统一基线 + 单因素改动 + 多随机种子 + 同延迟约束”为硬门槛，才能把 +0.002~0.005 级别提升从偶然波动中区分出来。

---

## 2. 突破点（创新点）合理性分析

### 2.1 合理且可落地的点

1. **“训练期复杂、推理期不增时延”的上下文建模路线（重参数化）**

- `SC-ELAN` 将多分支/大感受野（类 Pzconv）用于训练期特征学习，并在推理期折叠为单分支卷积，符合小目标对上下文敏感的动机，同时避免推理阶段额外分支开销。
- 这条路线在工程上具备可复用性：若收益明确，可在不改变部署接口的前提下推广到更多骨干块。

2. **“强上下文主干 + 选择性门控”的结构主线与证据一致**

- 文档的阶段总结与 Key Findings 指向：`LSKA`（长程空间建模）与 `TSCG`（选择性上下文门控）的组合是当前最有效结构基座。
- `k=23` 相比 `k=11` 的 LSKA 核尺度提升在严格指标（mAP50-95）与召回上均有持续增益（文档已归纳）。

3. **CAI（检测头重加权）的定位是合理的：不改推理“契约”，尝试提升长尾小类**

- `DetectCAI-Soft` 的价值不在于顶线更高（与 `LSKA23-TSCG` 并列），而在于“顶线不掉的前提下对 `pedestrian/people/bicycle/tricycle` 等小目标/长尾类有更明确的收益”。
- 这类“尾部收益不牺牲整体”的故事更适合论文叙事，但前提是稳定可复现。

### 2.2 当前“创新叙事”的薄弱环节（需要补齐）

1. **增益幅度小，必须用更严格的消融闭环证明“不可替代性”**

- 顶线从 `YOLO11-SCELAN` 的 **0.203** 推到 val4 **0.210**，属于可见但有限的增量。
- 这意味着：任何“多模块组合”都必须回答“是不是换个注意力/换个门控也行？”否则容易被审稿人判定为“工程堆叠”。

2. **功能重叠与边际收益递减风险显著**

- `LSKA`、`TSCG`、以及 CAI 的不同形态都在做“上下文强化/特征选择/重加权”，存在互相覆盖的可能。
- 文档也承认精度与速度存在 Pareto 前沿；继续叠加模块更可能把点推向“更慢但不更准”。

3. **CAI 稳定性是首要风险点（必须约束探索空间）**

- `Soft` 能达到并列顶线，`Mom098` 回退，说明 CAI 并非普适增益，且对策略超参敏感。
- 若缺少多随机种子与同协议对比，很难区分“真实提升”与“偶然波动/训练噪声”。

## 3. 下一步研究方向约束（硬约束 + 允许的探索边界）

> 目标：把探索空间收敛成“可发表、可复现、可解释”的路线，避免无效扩散。

### 3.1 统一主线与基线（不允许漂移）

- **固定结构基座（二选一）**：所有新实验必须从以下之一出发，仅做单因素修改。
  - `LSKA23-TSCG (val4)`：结构顶线、召回强、严格定位强（文档推荐 “Best Overall”）。
  - `LSKA-TSCG + DetectCAI-Soft (val4)`：顶线并列且长尾小类收益更明确（文档推荐 “Balanced / follow-up”）。
- **统一对比协议**：
  - 使用同一训练设置（seed、epochs、增强、优化器、batch 等）进行对比，任何偏离必须显式记录并单独归因。
  - 报告必须包含：整体 `P/R/mAP50/mAP50-95` + 指定小目标/长尾类（至少 `people/bicycle/tricycle`，可附 `pedestrian`）的逐类变化。

### 3.2 硬门槛 KPI（不满足即停止该方向）

- **主要目标（顶线）**：整体 `mAP50-95` 必须 **> 0.210**（当前基准：val4 `LSKA23-TSCG` 与 `DetectCAI-Soft` 并列 0.210）。
- **延迟约束（同延迟原则）**：
  - 总延迟必须维持在文档约束的实时范围（RTX 4090 约 **4.5–5.8 ms/图像**）内；
  - 若超出该范围，则需要提供“同延迟裁剪版/对照版”证明收益来自结构而非算力堆叠，否则不进入主线。
- **车辆类稳定性约束**：在提升 `people/bicycle/tricycle` 的同时，不允许车辆类（如 `car/van/bus`）出现系统性明显回退；若出现，必须提供原因诊断与修复策略，否则停止合并。

### 3.3 统计与评估协议约束（把小增益变成可信结论）

- **多随机种子**：任何宣称“提升”的变体必须至少做多 seed 报告（均值 ± 标准差）。  
  对于 +0.002~0.005 级别提升，没有方差报告一律视为“不确定收益”。
- **统一基准卡**：每次记录必须维护同一张表（精度、延迟、GFLOPs、参数量、显存、导出状态）。缺任何一项不算完成一次实验闭环。
- **跨数据集验证门槛**：任何宣称“突破 0.210”的候选，必须在至少 1 个额外无人机/交通风格数据集上复核趋势，否则仅能标注为“VisDrone 特化结果”，不用于模块定型与论文主结论。

## 4. 允许的研究方向（收敛后的“探索菜单”）

> 限制：每条方向一次只允许改动一个变量；改动前后必须可解释、可回滚、可复现。

### 4.1 方向 A：CAI 受控消融扫描（只允许在“Soft 邻域”内探索）

- **允许改动**：`cai_alpha/cai_beta`（以及与 Soft 相关的平滑/重加权强度），一次只动一个超参或一组联动超参（需说明联动理由）。
- **禁止改动**：
  - 直接跳到激进 EMA 类方案（文档已给出 `Mom098` 回退案例），除非先给出理论解释与稳定性验证计划。
  - 同时修改 backbone/neck 结构与 CAI（否则不可归因）。
- **验收标准**：
  - 顶线不低于基座，且 `people/bicycle/tricycle` 至少两类有稳定提升；
  - 多 seed 下优势仍存在（不要求每个 seed 都赢，但均值提升且方差可接受）。

### 4.2 方向 B：LSKA 核尺寸“阶段调度”（只允许在 7/11/23 之间）

- **允许改动**：按 stage 或按深度调度 `k_size`（例如浅层小核、深层大核），明确切换位置与理由。
- **禁止改动**：同时改 TSCG/CAI；否则无法证明提升来自“核尺寸调度”。
- **验收标准**：在不突破延迟约束的前提下，`mAP50-95` 有统计意义提升或在同延迟下取得更优点。

### 4.3 方向 C：TSCG “阶段感知门控”（限定为门控策略，不做新注意力堆叠）

- **允许改动**：门控的阶段开关、门控强度/阈值/路由规则（必须可解释）。
- **禁止改动**：引入新的大型注意力模块与额外分支（避免把问题退化为“继续叠模块”）。
- **验收标准**：提升必须体现在 mAP50-95（严格定位），而不仅是 mAP50；并且不引入明显的速度回退。

## 5. “停止规则”（避免无效内耗）

满足任一条，即停止该方向或降级为“备选/补充”：

- 连续 2 轮受控消融后，`mAP50-95` 无法稳定超过 0.210，且逐类收益不稳定；
- 提升只出现在 `mAP50`，而 `mAP50-95` 不升或下降（说明定位质量未改善）；
- 延迟或 GFLOPs 增长超过约束，但同延迟对照无法证明收益；
- 多 seed 下均值不升或方差过大，无法支撑“可复现提升”的结论。

---

## 6. 建议的最小可行实验清单（MVP）

> 目的：用最少实验量拿到“可发表证据链”，而不是最大化试错。

1. **基座确认（复测）**

- 复跑 `LSKA23-TSCG` 与 `DetectCAI-Soft` 至少 3 seeds，形成均值 ± 标准差的顶线卡片。

2. **优先级 A：CAI Soft 邻域扫描**

- 在 `DetectCAI-Soft` 基础上，对 `cai_alpha/cai_beta` 做小范围网格/步进（每次只动一项），每个点至少 2~3 seeds。

3. **优先级 B：LSKA 调度（同延迟约束）**

- 在 `LSKA23-TSCG` 基础上，尝试 1~2 种阶段调度策略（例如仅深层使用 k=23），确保延迟仍在约束内。

4. **泛化门槛：跨数据集复核**

- 对“突破 0.210 的候选”在额外数据集复核趋势（只需验证方向一致性与不崩溃，不要求绝对值对齐）。

---

## 7. 你可以直接复用的记录模板（推荐）

每条实验至少记录以下字段（可与文档的“统一基准卡”对齐）：

- 变体名 / YAML / commit hash
- 训练设置（epochs、imgsz、batch、optimizer、lr、aug、seed）
- 指标（all：P/R/mAP50/mAP50-95；指定类：people/bicycle/tricycle/pedestrian 的 mAP50 与 mAP50-95）
- 复杂度（Params、GFLOPs）
- 速度（pre/infer/post/total ms，硬件与 batch 设置）
- 导出状态（是否可导出、是否满足推理期结构约束）
