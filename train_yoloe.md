# YOLOE-26 — Data & Training Reference

Branch `yoloe_new` · entry point `train_yoloe26.py` · runs on **ultra6**.

---

## 1. Data on ultra6

Pick a dataset with the `--data <key>` flag. All active configs use absolute paths
under the shared NFS mount `/data/shared-datasets/yoloe26_data/` (portable across
all ultra6 users).

### A. "Old engine" data — made by Louis

`old_engine_data` reads **`.engine.cache`** label files (a cached label format),
under `/data/shared-datasets/yoloe26_data/`.

| set | images | labels (`.engine.cache`) |
|-----|--------|--------------------------|
| flickr | `/data/shared-datasets/yoloe26_data/flickr/full_images/` | `/data/shared-datasets/yoloe26_data/flickr/annotations/final_flickr_separateGT_train_segm.engine.cache` |
| mixed_grounding | `/data/shared-datasets/yoloe26_data/mixed_grounding/gqa/images` | `/data/shared-datasets/yoloe26_data/mixed_grounding/annotations/final_mixed_train_no_coco_segm.engine.cache` |
| Objects365v1 | `/data/shared-datasets/yoloe26_data/Objects365v1/images/train` | `/data/shared-datasets/yoloe26_data/Objects365v1/annotations/objects365_train_segm.engine.cache` |
| **val** (lvis) | `/data/shared-datasets/louis_data/lvis.yaml` | — |

`--data` key: `old_engine_data` (flickr + mixed_grounding + Objects365v1).

> **Note.** An earlier version (commented out just above the active definition in
> `train_yoloe26.py`) used `/data/shared-datasets/louis_data/…` paths with raw
> `.json` labels. It's kept for reference only.

### B. "Enterprise" data — generated by Fatih's pipeline (v4 / v5)

Root: `/data/shared-datasets/yoloe26_data/` — images **and** labels both under this root.

| set | images | labels (json) |
|-----|--------|---------------|
| flickr | `…/yoloe26_data/flickr/full_images/` | `flickr/pipeline_outputs/{v4,v5}/merged.json` |
| mixed_grounding | `…/yoloe26_data/mixed_grounding/gqa/images` | `mixed_grounding/pipeline_outputs/{v4,v5}/merged.json` |
| Objects365v1 | `…/yoloe26_data/Objects365v1/images/train` | `Objects365v1/pipeline_outputs/train/{v4,v5}/merged.json` |
| yolo-enterprise | `…/yoloe26_data/yolo-enterprise/images/train` | `yolo-enterprise/pipeline_outputs/train/v4/merged.json`  (v5: `…/train/v5/merged_simplified.json`) |
| **val** (lvis) | `/data/shared-datasets/louis_data/lvis.yaml` | — |


The commands in section 2 use `--data old_engine_data`. To train on enterprise
data instead (v4 public + v5 enterprise), swap to `--data yedata`.

## 2. Training commands

Four stages; **VP / PF / SEG each fine-tune from the TP stage's `best.pt`.**

| stage | trainer | output dir | init weights |
|-------|---------|-----------|--------------|
| **TP** — text-prompt, from scratch | `YOLOETrainerFromScratch` | `runs/yoloe26_tp` | `yolo26{n,s,m,l,x}-objv1-150.pt` |
| **VP** — visual-prompt | `YOLOEVPTrainer` | `runs/yoloe26_vp` | TP `best.pt` |
| **PF** — prompt-free | `YOLOEPEFreeTrainer` | `runs/yoloe26_pf` | TP `best.pt` |
| **SEG** — segmentation head | `YOLOESegTrainerSegHead` | `runs/yoloe26_seg` | TP `best.pt` |

For long detached runs, prefix `nohuprun`.

**Before running:**

- **VP / PF / SEG** `--weight_path` must point at the real TP run directory. The
  host tags (`[]`, `[ultra6]`, `[ultra8]`) in the run names have drifted, so
  confirm the path exists on disk first.
- **TP-l / TP-x** add `--override` (`or260107_forl` / `or260105` = Jing's tuned
  hparams), which **replace** the `lr` / augmentation flags written on those two lines.
- `--o2m` is omitted (not a valid flag in the current `train_yoloe26.py`).

### Stage 1 — TP (text-prompt, from scratch).  Produces the base best.pt.

#### tp_n

```bash
python train_yoloe26.py \
  --model_version 26n --weight_path yolo26n-objv1-150.pt \
  --trainer YOLOETrainerFromScratch  \
  --data old_engine_data \
  --optimizer MuSGD --lr0 0.00125 --lrf 0.5 --momentum 0.9 --weight_decay 0.0005 \
  --epochs 30 --close_mosaic 2 --batch 256 --copy_paste 0.1 --mixup 0.0 \
  --device 0,1 \
  --project runs/yoloe26_tp \
  --name "26n_ptwobjv1_bs256_epo30_close2_engine_old_engine_data_tp1"
```
