# DukeMTMC-reID Dataset

The [DukeMTMC-reID](https://arxiv.org/abs/1701.07717) dataset is a large-scale person re-identification benchmark derived from the DukeMTMC multi-target multi-camera tracking dataset. It was collected on the Duke University campus using 8 synchronized cameras. The dataset contains 36,411 bounding boxes of 1,404 identities, making it one of the most popular ReID evaluation benchmarks alongside [Market-1501](market1501.md).

## Key Features

- **1,404 identities** captured by 8 cameras across the Duke University campus.
- **16,522 training images** from 702 identities.
- **2,228 query images** from 702 identities for evaluation.
- **17,661 gallery images** from 702 identities (plus distractors) for matching.
- Images include person bounding boxes with varied occlusion, pose, and illumination.
- Standard evaluation protocol excludes same-person-same-camera matches.

## Dataset Structure

The DukeMTMC-reID dataset is organized into three main directories:

1. **bounding_box_train/**: 16,522 images of 702 identities for training.
2. **query/**: 2,228 images used as queries during evaluation.
3. **bounding_box_test/**: 17,661 gallery images used for matching against queries.

### Filename Convention

Each image follows the naming pattern:

```
PPPP_cC_fXXXXXX.jpg
```

- `PPPP`: 4-digit person ID (e.g., `0001`)
- `C`: Camera ID (1-8)
- `XXXXXX`: Frame number

For example, `0001_c1_f0046182.jpg` means person `0001` captured by camera `1` at frame 46182.

## Benchmark Results

DukeMTMC-reID results from fine-tuning the `yolo26{size}-reid.pt` seed on DukeMTMC-reID (imgsz=448, standard query–gallery protocol):

| Model                     | mAP  | Rank-1 | Rank-5 | Rank-10 |
| ------------------------- | ---- | ------ | ------ | ------- |
| YOLO26n-reid (fine-tuned) | 48.0 | 69.8   | 82.5   | 86.4    |
| YOLO26l-reid (fine-tuned) | 57.0 | 75.8   | 86.3   | 89.5    |

With [k-reciprocal re-ranking](../../tasks/reid.md) (`reid_reranking=True`), mAP rises to 67.8 (n) and 74.3 (l). Larger seeds transfer better to this domain; see the [ReID Fine-Tuning guide](../../guides/reid-finetuning.md).

## Applications

DukeMTMC-reID is widely used for:

- **Cross-dataset evaluation** paired with Market-1501 for domain adaptation studies
- **Metric learning** and embedding space optimization
- **Occlusion-robust ReID** due to its diverse viewpoints and occlusion patterns
- **Unsupervised and semi-supervised** ReID method development

## Usage

To train a YOLO ReID model on DukeMTMC-reID for 60 epochs with an image size of 448, you can use the following code snippets. For a comprehensive list of available arguments, refer to the model [Training](../../modes/train.md) page.

!!! example "Train Example"

    === "Python"

        ```python
        from ultralytics import YOLO

        # Load a model
        model = YOLO("yolo26n-reid.yaml")

        # Train the model
        results = model.train(data="DukeMTMC-reID.yaml", epochs=60, imgsz=448)
        ```

    === "CLI"

        ```bash
        # Start training from a YAML model config
        yolo reid train data=DukeMTMC-reID.yaml model=yolo26n-reid.yaml epochs=60 imgsz=448
        ```

## FAQ

### How does DukeMTMC-reID compare to Market-1501?

DukeMTMC-reID has 8 cameras (vs 6 in Market-1501), more outdoor scenes, and greater viewpoint variation. It is often used alongside Market-1501 for cross-dataset transfer evaluation. Models typically achieve slightly lower mAP on DukeMTMC-reID due to its more challenging conditions.

### How do I set up DukeMTMC-reID for training?

1. Download the dataset from available mirrors or the original project page.
2. Extract the archive to your datasets directory.
3. The directory structure should contain `bounding_box_train/`, `query/`, and `bounding_box_test/` folders.
4. Train using `data=DukeMTMC-reID.yaml` which points to the standard dataset layout.
