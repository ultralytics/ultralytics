# LUPerson-NL Dataset

[LUPerson-NL](https://github.com/DengpanFu/LUPerson-NL) is a large-scale person [re-identification](https://www.ultralytics.com/glossary/reid-re-identification) **pre-training** dataset with noisy (tracker-generated) identity labels. It contains **10,683,716 images across 433,997 identities** mined from internet videos, making it orders of magnitude larger than supervised benchmarks such as [Market-1501](market1501.md) or [MSMT17](msmt17.md). In the Ultralytics ReID pipeline it serves as the **Step-1 supervised pretrain** that shapes a strong, general embedding space before fine-tuning on smaller labeled datasets.

## Key Features

- **10.68M images / 434K identities** in folder-per-identity layout — one of the largest ReID corpora available.
- **Noisy labels** generated by tracking, not manual annotation — large scale traded against label precision.
- Intended as a **pretrain stage**, not a train-from-scratch or evaluation target.
- Drives the released `yolo26{n,s,m,l,x}-reid.pt` seeds that you fine-tune on your own data.

!!! note "Pretrain, then fine-tune"

    LUPerson-NL is used with `val=False` — there is no meaningful query/gallery evaluation on the noisy labels. After pretraining, fine-tune on a labeled dataset ([Market-1501](market1501.md), [DukeMTMC-reID](dukemtmc.md), [MSMT17](msmt17.md), or your own). Most users should simply **start from the released `yolo26*-reid.pt` checkpoints**, which already include this pretrain — see the [ReID Fine-Tuning guide](../../guides/reid-finetuning.md).

## Dataset Structure

LUPerson-NL is stored folder-per-identity, where each subdirectory name is a (noisy) person ID:

```
LUPerson-NL/
├── train/                  # train/<person_id>/<key>.jpg — 10.68M images / 434K identities
│   ├── 0000001/
│   │   ├── ....jpg
│   │   └── ....jpg
│   └── ...
└── val_tiny/               # 8-identity placeholder so the trainer skips scanning all 10.68M images
```

Because the source keys do not encode a stable camera ID, `camid` is auto-assigned (unique per image). This only matters for the eval-time same-camera exclusion, which pretraining skips (`val=False`).

## Download and Build

The built-in `LUPerson-NL.yaml` includes an automatic download/build script that runs when the dataset is missing. It is a heavy, two-stage, I/O-bound job and requires `huggingface_hub` + `pyarrow`, Hugging Face authentication for the mirror, and fast NVMe storage:

1. `snapshot_download` the Hugging Face Arrow shards (mirror `wahdan2003/LUPerson-NL`) into `<path>/hf`.
2. Convert the Arrow shards to `train/<person_id>/<key>.<ext>` (folder-per-identity, raw bytes, no re-encode).
3. Copy 8 training identities into `val_tiny/` as a placeholder validation loader.

!!! warning "Large download"

    The full corpus is ~10.68M images / 434K identities. Ensure you have ample disk and bandwidth before triggering the build. Point `path` at a location with enough space (or symlink the dataset root to where the data actually lives).

## Applications

LUPerson-NL is used for:

- **Large-scale ReID pre-training** that generalizes across downstream ReID datasets.
- **Noisy-label learning** research at scale.
- Building **general-purpose embedding seeds** for transfer to person and non-person domains.

## Usage

To run the Step-1 pretrain on LUPerson-NL (validation disabled), use the snippets below. For a comprehensive list of available arguments, refer to the model [Training](../../modes/train.md) page.

!!! example "Pretrain Example"

    === "Python"

        ```python
        from ultralytics import YOLO

        # Build a ReID model from a YAML config
        model = YOLO("yolo26l-reid.yaml")

        # Step-1 supervised pretrain on LUPerson-NL (no validation)
        results = model.train(data="LUPerson-NL.yaml", val=False, imgsz=256)
        ```

    === "CLI"

        ```bash
        # Step-1 supervised pretrain on LUPerson-NL
        yolo reid train data=LUPerson-NL.yaml model=yolo26l-reid.yaml val=False imgsz=256
        ```

## FAQ

### Do I need to download LUPerson-NL to use YOLO ReID?

No. The released `yolo26*-reid.pt` checkpoints are already pretrained, so most users should fine-tune directly from them. Download LUPerson-NL only if you want to reproduce or customize the Step-1 pretrain.

### Why is validation disabled (`val=False`) for LUPerson-NL?

The identity labels are tracker-generated and noisy, so a query/gallery evaluation on them is not meaningful. The dataset is a pretrain stage; evaluation happens after fine-tuning on a labeled benchmark.

### How is LUPerson-NL different from MSMT17 as a pretrain?

[MSMT17](msmt17.md) is the largest **labeled** benchmark (~126K images), whereas LUPerson-NL trades label precision for scale (~10.68M images). It replaces or augments the smaller MSMT17 pretrain to produce a stronger general embedding before fine-tuning.
