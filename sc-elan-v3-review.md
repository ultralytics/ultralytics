# SC-ELAN v3 设计评审（DetectCAIv3）

## 1. 当前情况（截至 2026-02-28）

基于 `sc-elan.md` 的现有结论，当前系统状态可以归纳为：

- 最优严格指标来自 `v2-P1d`：`mAP50-95 = 0.212`，`mAP50 = 0.367`，`P = 0.490`，`R = 0.381`。
- 当前最有效增益来源是 **CAI 参数调优**，不是继续增加结构复杂度。
- 在相同骨干下，`alpha=0.10`、更高 `beta`（`0.25 -> 0.40`）带来了明确提升（`0.209 -> 0.212`）。
- Phase 2（SA-LSKA/TSCGv2）与 Phase 3（P3-FRM）未超越 `v2-P1d`，说明“结构继续堆叠”的边际收益已下降。
- 困难类别仍集中在 `people / bicycle / tricycle`，其中 `people` 提升最受限。
- 延迟约束明确：需维持在当前实时区间（约 `5.6–5.9 ms`，RTX 4090）。

结论：我们已进入“强主干基本稳定，头部训练机制是主要优化杠杆”的阶段。

## 2. 为什么需要这次修改

`v2-P1d` 的成功是强信号，但也暴露出固定参数 CAI 的局限：

1. **固定 `beta` 是全局一刀切**  
   不同图像、不同尺度、不同类别难度下，理应需要不同的增强强度。固定 `beta=0.40` 不能表达这种条件差异。

2. **高增益与稳定性存在张力**  
   文档中 `Mom098` 的退化结果说明：重加权策略一旦过激，可能导致全局回退。当前体系缺少显式稳定器（如暖启动、硬边界）。

3. **研究方向需要“更强逻辑可解释性”**  
   如果后续目标是可发表的模块创新，必须回答：  
   “增强何时发生、为什么发生、为何不会失控”。  
   固定 CAI 难以给出严密机制解释。

因此，本次修改不是换主干，而是在 `P1d` 成功机理上做**机制升级**：  
从“固定增强”升级为“条件化、有界、可解释的增强”。

## 3. 修改内容概述

本次新增 `DetectCAIv3`（训练期生效，推理路径不变）：

- 继承 `DetectCAI`，保留 `alpha=0.10`、`beta=0.40` 的有效基线思想。
- 新增 **动态 beta 路由**，以三类信息调节增强强度：
  - tail prior（长尾先验强度）
  - detail score（局部高频细节强度）
  - uncertainty（分类不确定度）
- 新增 **warmup**：前期逐步开启重加权，降低训练早期震荡。
- 新增 **gate clipping**：限制门控范围，防止过放大/过抑制。
- 支持 **level gain**（按金字塔层级设置系数），让 P3/P4/P5 可采用不同上下文注入强度。

对应代码与配置：

- `ultralytics/nn/modules/head.py`：新增 `DetectCAIv3`
- `ultralytics/nn/tasks.py`：模型解析器注册 `DetectCAIv3`
- `ultralytics/nn/modules/__init__.py`：导出 `DetectCAIv3`
- `models/sc_elan/yolo11-scelan-v3-p1d-adacai.yaml`：新实验配置

## 4. 设计逻辑（严密性）

### 4.1 目标函数层面的直觉

`P1d` 说明“适度基础增益 + 较强 tail 交互增益”有利于严格定位指标。  
`v3` 的逻辑是：保留该方向，但把“强度”从常数改为受样本状态驱动的变量。

### 4.2 动态强度的构造

通过以下因子构造 `beta_dyn`：

- tail 强：说明当前分布更偏困难类别，应提高交互增益。
- detail 强：说明特征中存在更明显的小目标细节，应适度提高增益。
- uncertainty 高：说明分类判别不稳，应允许更强条件校准。

并通过上下界裁剪，保证行为稳定、可控。

### 4.3 稳定性保障

- warmup 防止模型初期被强重加权扰动。
- gate clipping 抑制长尾极端场景下的数值发散。
- 保留推理时与 Detect 等价的部署路径，确保工程可落地。

## 5. 预期收益

1. 在不改主干推理结构的前提下，争取超越 `0.212` 的严格指标。
2. 对 `people / bicycle / tricycle` 形成更有针对性的增益。
3. 提高实验解释性：可分析“何时增强、增强多少、在哪一层增强”。
4. 更贴合当前阶段目标：优先优化头部训练机制，而不是继续堆结构。

## 6. 风险与应对

- 风险 1：动态路由噪声过大，导致训练波动  
  - 应对：先收紧 `beta_ratio` 和 `gate` 上下界，再观察曲线稳定性。

- 风险 2：tail 提升换来主流类别回退  
  - 应对：按类追踪并对 `gamma_tail` 做小步退火。

- 风险 3：指标提升不显著  
  - 应对：优先做参数归因消融（仅 warmup、仅 clipping、仅 dynamic beta），明确瓶颈位置。

## 7. 评估策略（按当前要求：不执行 seeds）

先进行单种子对比评估：

1. Baseline：`v2-P1d`
2. New：`yolo11-scelan-v3-p1d-adacai`
3. 其余训练设置保持一致

报告项：

- 全局：`P / R / mAP50 / mAP50-95`
- 难类：`people / bicycle / tricycle`
- 代价：延迟、GFLOPs、参数量
- 稳定性：loss 是否尖峰、是否出现 NaN、门控是否饱和

当前阶段不做多 seeds，待单次结果确认有效后再决定是否进入统计验证。

## 8. 评审结论

该修改是一次“沿着 P1d 成功机理做机制化升级”的合理演进，而非盲目增加复杂度。  
它在理论上更可解释、在工程上保持低部署风险、在研究上具备可消融验证的清晰路径。  
在当前证据下，`DetectCAIv3` 是值得进入下一轮主线实验的方案。
